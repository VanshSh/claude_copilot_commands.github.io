Slash Commands
In-session commands — type these at the Claude Code prompt
/help
essential
Shows all available slash commands including custom commands from your .claude/commands/ directory and connected MCP servers.
/help
/compact
essentialperf
Compresses conversation history into a dense summary, freeing context window space. Pass instructions to control what gets retained. Use when context usage exceeds 80%.
/compact /compact Focus on the auth module and current test failures
/clear
essential
Hard reset — wipes conversation history entirely and frees all context. Also aliased as /reset and /new. Use at task boundaries when switching to something unrelated.
/clear
/init
essentialconfig
Analyzes your project and generates a CLAUDE.md file containing project-specific rules, conventions, and context that persists across all sessions.
/init
/model
essentialperf
Switch AI models mid-session without losing conversation. Use Sonnet for exploration and file reading; Opus for complex problems. Switching reduces token costs significantly.
/model sonnet /model opus /model haiku # cheapest, fast tasks
/diff
essential
Opens an interactive diff viewer showing all changes made during the session. Toggle between full git diff and per-turn diffs, navigate between files with arrow keys.
/diff
/cost
perf
Shows API token usage and cost for the current session (API users). Pro/Max subscribers use /stats instead. Helps build intuition for expensive tasks.
/cost /stats # Pro/Max subscribers /context # check context window fullness /usage # check rate limit proximity
/plan
advanced
Puts Claude in read-only mode. It can read and analyze but cannot make changes — all proposed modifications require explicit approval. Also toggled via Shift+Tab. Use before touching production-critical files.
/plan # also Shift+Tab to toggle
/resume
essential
Picks up a previous session. Without arguments opens a session picker. With a name, jumps directly there. Alias: /continue. CLI equivalent: claude -c (last), claude -r "name" (by name).
/resume /resume auth-refactor # From terminal: claude -c claude -r "auth-refactor"
/rename
essential
Gives the current session a meaningful name. Without arguments, auto-generates one from context. Makes sessions findable weeks later.
/rename auth-refactor-2026 /rename # auto-generate from context
/fork
advanced
Creates a branch of the current conversation. Explore one approach, then /resume back to the fork point and try something different. The conversational equivalent of a git branch.
/fork # branch here, experiment safely # If approach fails: /resume # back to fork point
/rewind
advanced
Rolls back conversation to a previous point. Selective rollback lets you revert all file changes while keeping conversation history. Alias: /checkpoint. Also triggered by double-tap Esc.
/rewind /checkpoint # Esc+Esc → choose "Rewind code only"
/export
advanced
Exports the current conversation as plain text. Useful for documentation, post-mortems, or sharing a session's output with a teammate.
/export
/insights
advanced
Reads your past month of usage and compiles a detailed HTML report covering usage patterns, costs, tool usage, and session history. Run monthly to improve your workflow.
/insights
/security-review
advanced
Analyzes changes on your current git branch specifically for security vulnerabilities. Scoped and targeted — run before any PR touching authentication, authorization, or data handling.
/security-review
/simplify
advanced
Runs a three-agent review pipeline checking for architectural issues, duplicate logic, and performance inefficiencies. Use as a quality gate before PRs, not during active development.
/simplify
/batch
advanced
Runs large-scale changes across multiple files in parallel using isolated git worktrees, then automatically creates pull requests. Great for applying a pattern change across 50+ files.
/batch rename all occurrences of UserID to user_id across src/
/btw
advanced
Asks a side question without it becoming part of the main conversation thread. Claude answers, then the thread continues from where it was. Keeps context clean during deep debugging.
/btw what's the difference between PUT and PATCH?
/memory
config
Opens the CLAUDE.md editing interface for adding, updating, or removing project rules. Invest 10 minutes after setup — pays back in every subsequent session.
/memory
/hooks
configadvanced
Manages automation that triggers on tool events: before file edits, after bash commands, on session start. Hooks guarantee execution (unlike prompts). Essential for enforcing linting, formatting, CI rules.
/hooks
/permissions
config
Controls which tools Claude can use and what file paths are permitted. Essential for CI/CD pipelines and security-sensitive environments where you need to restrict the blast radius.
/permissions /allowed-tools
/agents
advanced
Manages subagent configurations — specialized Claude instances with their own context windows, personas, and tool access. Delegate complex tasks (code review, debugging, architecture) to focused subagents.
/agents
/remote-control
advanced
Enables remote control of your local Claude Code session from claude.ai. Alias: /rc. Monitor long-running tasks, send instructions, and review results from any device. Requires Claude Max.
/remote-control /rc # shorthand
/terminal-setup
config
Configures your terminal for full keyboard shortcut support. Run this once after installation to activate Alt+T (extended thinking toggle) and other keyboard shortcuts.
/terminal-setup
/extra-usage
perf
Configure what happens when you hit your rate limit. Useful for Pro/Max subscribers who need to manage usage carefully near the end of their billing cycle.
/extra-usage

CLI Flags
Flags passed when launching Claude Code from the terminal
--print / -p
essential
Sends a query and exits without starting an interactive session. The gateway to scripting. Pair with --output-format json for machine-readable output in CI/CD pipelines.
claude -p "Explain this repository" > overview.txt claude --print "summarize changes" --output-format json
--output-format
advanced
Controls output format when using print mode. Options: text (default), json (machine-readable), stream-json (streaming JSON). Essential for scripting and automation.
claude -p "review code" --output-format json claude -p "analyze" --output-format stream-json
--resume / -c
essential
Resume the most recent session (-c) or resume by session name (-r "name"). Never lose context when switching tasks.
claude -c # resume last session claude -r "auth-refactor" # resume by name
--allowedTools
advanced
Limits which tools Claude can use in a session. For automated analysis where you want read-only behavior. Multiple tools can be specified space-separated.
claude -p --allowedTools "Read" "Grep" "Glob" \ "Find issues in this codebase"
--max-turns
advancedperf
Caps the number of agentic turns before stopping. A circuit breaker for automated pipelines where runaway execution would be costly. Essential for CI/CD.
claude -p "fix all lint errors" --max-turns 10
--worktree / -w
advanced
Creates an isolated git worktree and starts a session inside it. Changes are isolated from your main branch. Worktrees are auto-cleaned if no changes are made. Used internally by /batch.
claude --worktree claude -w
--dangerously-skip-permissions
advanced
YOLO mode (v2.0.68+). Fully autonomous operation — skips all permission prompts. Use only in isolated/sandboxed environments. Never on production systems.
claude --dangerously-skip-permissions \ "refactor all component files"
--system-prompt
config
Override the system prompt entirely. Mutually exclusive with --system-prompt-file. Use --append-system-prompt to add to existing prompt instead of replacing it.
claude --system-prompt "You are a security-focused code reviewer" claude --append-system-prompt "Always suggest tests" claude --system-prompt-file ./prompts/reviewer.txt
--agents
advanced
Defines custom subagents inline at launch without requiring configuration files. Useful for CI/CD pipelines where you want to define agent behavior in the pipeline config itself.
claude --agents reviewer:./agents/reviewer.md
--verbose / --debug
config
Enable verbose logging and debug output. Useful for troubleshooting authentication issues, tool execution failures, or unexpected behavior.
claude --verbose claude --verbose --debug claude "test" --verbose
--teleport
advanced
Pulls a remote Claude Code session (from claude.ai/code) to your local terminal. Get full conversation history, modified files, and continue the interaction locally. Requires Claude Max.
claude --teleport session-id-from-web
--vscode
config
Open Claude Code integrated with VS Code. Enables the IDE integration features including inline diffs, file navigation, and the Claude Code extension panel.
claude --vscode

CLI Flags
Flags for launching and scripting Copilot CLI
--prompt / -p
essential
Send a single prompt and exit (programmatic mode). Essential for scripting and CI/CD pipelines. Combine with approval flags to allow file modifications.
copilot -p "summarize changes in this PR" copilot --prompt "fix lint errors" --allow-all-tools
--allow-all-tools
advanced
Allow all tools without manual approval in programmatic mode. Grants Copilot the same file and shell access you have. Use with caution; only in trusted environments.
copilot -p "update dependencies" --allow-all-tools
--allow-tool / --deny-tool
advanced
Fine-grained tool permission control. Accept patterns in format Kind(argument). Deny rules always take precedence over allow rules, even when --allow-all is set.
copilot --allow-tool='shell(git:*)' \ --deny-tool='shell(git push)' # Allow git read, block destructive push
--experimental
new
Launch with experimental features enabled. Once set via /experimental on in a session, the flag is no longer needed on subsequent launches.
copilot --experimental
--acp
advanced
Start as an Agent Client Protocol (ACP) server. Allows third-party tools, IDEs, and automation systems to interact with Copilot's agentic capabilities via the open ACP standard.
copilot --acp copilot --acp --port 3000 # TCP mode
--additional-mcp-config
config
Add MCP servers for a single session without modifying persistent config. Useful for session-specific integrations or testing new MCP server configurations.
copilot --additional-mcp-config ./mcp-test.json
--disable-builtin-mcps
config
Disable all built-in MCP servers. Use --disable-mcp-server SERVER-NAME to disable a specific one. Useful for controlled/sandboxed environments.
copilot --disable-builtin-mcps copilot --disable-mcp-server github
--plugin-dir
config
Specify a custom directory for loading plugins. Useful for development or testing custom plugins before publishing to a marketplace.
copilot --plugin-dir ./my-plugins
--share / --share-gist
advanced
Export session on completion. --share saves locally, --share-gist creates a GitHub Gist. Includes timestamp, model, token usage, and full conversation.
copilot -p "analyze codebase" --share copilot -p "review PR" --share-gist

Configuration Files
Copilot CLI customization hierarchy — settings cascade from user to repository to local
File / Location Scope Purpose
.github/copilot-instructions.md Repo-wide Persistent guidance loaded at every session start. Coding standards, conventions, build commands, preferred patterns. Applied to every prompt automatically.
.github/instructions/**/*.instructions.md File-pattern Path-based instructions with applyTo glob in YAML frontmatter. Target TypeScript files, test directories, API routes, etc. Combines with repo-wide instructions.
AGENTS.md / CLAUDE.md / GEMINI.md Repo root Multi-agent instruction files at repo root. Root-level files are treated as primary instructions. Supports all major AI agents.
~/.copilot/copilot-instructions.md Global (user) Personal preferences across all repos. Coding style, communication preferences, tool choices. Also loaded via COPILOT_CUSTOM_INSTRUCTIONS_DIRS env var.
.github/skills/<n>/SKILL.md Project Skills with YAML frontmatter (name + description). Invoked with /skill-name or auto-detected when relevant. Open standard (agentskills spec).
~/.copilot/mcp-config.json Global Persistent MCP server configuration. GitHub MCP server is built-in. Add custom servers for DBs, APIs, calendars.
.github/hooks/*.json Repository Lifecycle hook configuration. Execute shell commands at sessionStart, sessionEnd, preToolUse, postToolUse, userPromptSubmitted, agentStop, errorOccurred.
.github/agents/ Repository Custom agent definitions in Markdown with YAML frontmatter. Define role, allowed tools, MCP connections. Auto-delegated when infer: true is set.
~/.copilot/agents/ Global User-level custom agent definitions used across all repos.
.github/copilot/settings.json Repository Repository-level Copilot settings. Committed to the repo and shared with the team.
.github/copilot/settings.local.json Local override Personal overrides for repository settings. Add to .gitignore — not committed. Same schema as settings.json but takes precedence.
~/.copilot/lsp-config.json Global Language Server Protocol configuration per language. Enables go-to-definition, hover info, diagnostics.

Token Optimization & Cost Reduction
Practical strategies to maximize speed, minimize cost, and get better results from both tools
📊
Cost Reality Check
A typical coding session with Claude Code consumes 50K–200K input tokens and 10K–50K output tokens. With Haiku that's $0.10–$0.45/session; with Opus the same session costs $0.50–$2.25 (5× more). With Copilot CLI, each interaction uses one premium request from your monthly allowance. GPT-4.1 and GPT-5 mini are free (no premium cost). Smart tool selection has a direct, measurable impact on your bill.
1
Choose the Right Model for Each Task
Don't use the most powerful model for every task. Use Haiku/Sonnet for exploration and file reading. Reserve Opus for genuinely complex problems. Switch back once the hard part is solved.
# Claude Code /model haiku # for simple file searches /model sonnet # for most coding tasks /model opus # only for hard problems # Copilot CLI /model gpt-4.1 # free, no premium cost /model claude-sonnet # balanced, best value /model claude-opus # complex architecture only
2
Compact Before Context Overflows
Don't wait for the "context is getting large" warning. Run /compact proactively at 80% context usage with focus instructions. Starting fresh after overflow wastes all accumulated context.
# Include focus instructions for better summary /compact Focus on the auth module, ignore UI files # Claude Code: check context level anytime /context # Copilot: auto-compacts at 95% in background # Manual compact anytime: /compact
3
Use ! for Direct Shell Commands
Prefixing a command with ! executes it directly, bypassing Claude's conversational mode. Claude only sees the final result, not a narrated description of what it's about to do. Fewer tokens, faster results.
# SLOWER: Claude narrates the action run npm test to check the current state # FASTER: Direct execution, just results !npm test !git status !ls src/components/ | grep Button
4
Reference Files Directly with @
Using @filename puts the exact file contents in context without Claude searching for it first. Eliminates several tool-use turns and gives Claude precise scope. Copilot also supports #issue for GitHub issues.
# SLOW: Claude searches for the file Fix the bug in the authentication component # FAST: Direct file reference @src/auth/LoginForm.tsx fix the form validation bug @src/api/auth.ts explain the token refresh logic # Copilot: also reference GitHub issues #123 implement the feature described here
5
Use CLAUDE.md / copilot-instructions.md
Invest time writing good project instructions once. Every prompt you repeat (your stack, conventions, test framework) is wasted tokens. Project memory means you never re-explain your codebase.
# Claude Code: generate and edit /init # auto-generate from codebase /memory # edit instructions # Copilot: create manually .github/copilot-instructions.md # Both: include in instructions - Tech stack and versions - Coding conventions and patterns - Test framework and how to run tests - Build commands - File naming conventions
6
Use Plan Mode Before Implementing
Plan mode is read-only — no file writes, no token-burning failed attempts. Get the plan right first. One good plan with your feedback is worth 3 wrong implementations that each require rollback and retry.
# Claude Code: plan before implementing /plan (or Shift+Tab) # Copilot: explicitly request a plan /plan implement OAuth2 with PKCE for mobile # Only after reviewing the plan: Shift+Tab → back to normal mode "Looks good, proceed with step 1"
7
Restrict Tools in Automated Pipelines
In CI/CD, don't give the AI read-write-execute permissions when you only need analysis. Read-only tool restrictions prevent accidental state changes and make token usage predictable.
# Claude Code: read-only CI analysis claude -p "find security issues" \ --allowedTools "Read" "Grep" "Glob" # Copilot: granular shell restrictions copilot --allow-tool='shell(git:*)' \ --deny-tool='shell(git push)' \ --deny-tool='shell(rm:*)' # Set max turns to prevent runaway loops claude -p "fix all bugs" --max-turns 15
8
Use Subagents for Parallelism & Focus
Complex tasks benefit from delegation. A "code-reviewer" subagent paired with a "debugger" agent produces better results than one generalist handling everything. Each subagent has its own focused context window.
# Claude Code: define subagents # .claude/agents/reviewer.md --- name: reviewer description: Use for code reviews model: sonnet --- You focus only on security and performance. # Copilot: use fleet for parallel tasks /fleet run the test suite while fixing the bug # Delegate long tasks to cloud agent & implement the full settings page
9
Monitor Costs in Real Time
A single agentic loop can burn through tokens faster than you'd expect. Build habits around checking usage. Know when a task is becoming expensive before it's too late to switch strategy.
# Claude Code: session health check /cost # API users: current spend /stats # Pro/Max: usage summary /context # how full is context window? /usage # near rate limit? # If costs climbing: /compact # compress context /model sonnet # switch to cheaper model # Run insights monthly /insights # 30-day usage HTML report
10
Use Skills & Custom Commands
Repeating the same complex prompt for common tasks is wasteful. Package them as skills or custom commands. One well-written skill invocation vs. a 5-sentence prompt every time adds up significantly.
# Claude Code: create a skill # .claude/skills/pr-review/SKILL.md --- name: pr-review description: Review current branch for PR --- Review all changes on this branch: 1. Check for security vulnerabilities 2. Check test coverage 3. Suggest improvements # Invoke: /pr-review # Copilot: skills in .github/skills/ # Invoke: /skill-name
11
Use Prompt Caching (API Users)
For teams using the Claude API directly: prompt caching gives 90% savings on cache reads. Cache writes cost 1.25× base (5-min) or 2× (1-hour), but cache reads cost only 0.1×. Structure prompts to maximize cache hits.
# Cache system prompts and large contexts # Keep static context at the FRONT of prompts # (dynamic user input at the end) # Minimum cacheable block: 1024 tokens # Copilot equivalent: use repository memory # Copilot learns patterns automatically and # reduces repetitive context across sessions
12
Leverage Extended Thinking Wisely
Extended thinking is powerful but costs more tokens. Don't use it for routine tasks. Reserve it for genuinely complex problems: hard architectural decisions, debugging tricky race conditions, complex algorithm design.
# Claude Code: toggle for current turn only Alt+T # (run /terminal-setup first) # Or use the toggle in settings (persists) # NOT recommended for: # - Simple file edits # - Boilerplate generation # - Answering quick questions # USE for: # - System architecture decisions # - Debugging complex race conditions # - Designing algorithms
