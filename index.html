SESSION MANAGEMENT

/clear
essential
aliases: /reset · /new
Wipe entire conversation history and start fresh. Also resets per-directory command history. Use at task boundaries when old context would confuse Claude. Unlike /compact it does not retain any summary.
/clear
/compact
essentialperf
optional: [focus instructions]
Compress conversation history into a dense summary to free context space. Pass optional instructions to control what gets retained. Use proactively when context exceeds 80%. Since v2.0.64 compaction completes instantly.
/compact /compact retain the auth flow and test failures /compact keep database schema, ignore UI changes
/resume
essential
alias: /continue
Resume a previous session. Without arguments opens a session picker. With a name or ID jumps directly there. CLI equivalents: claude -c (last session), claude -r "name" (by name).
/resume /resume auth-refactor # From terminal: claude -c / claude -r "auth-refactor"
/fork
advanced
optional: [name]
Branch the current conversation into a new session from this point. Explore one approach, then /resume back to the fork point and try another. The conversational equivalent of git branch — experiment without destroying working state.
/fork try-redis-approach # If it fails, resume back: /resume
/rename
optional: [name]
Rename the current session. Without a name, auto-generates one from conversation history. Named sessions are findable weeks later via /resume.
/rename payment-refactor-march /rename # auto-generate from context
/rewind
advanced
alias: /checkpoint · also: Esc+Esc
Restore code and conversation to a previous point. Selective rollback lets you revert all file changes while keeping conversation history. "Rewind code only" reverts files without losing the diagnostic conversation.
/rewind /checkpoint # Or: Esc+Esc → choose "Rewind code only"
/exit
alias: /quit
Exit Claude Code. Session is automatically saved and can be resumed later with /resume or claude -c.
/exit # or Ctrl+D, Ctrl+C twice

INFORMATION & DIAGNOSTICS

/help
essential
List all available commands including custom skills from .claude/commands/ and .claude/skills/, plus any commands from connected MCP servers. Type any letters after / to filter the list in real time.
/help
/cost
perf
Show token usage and cost for the current session (API users). A single agentic loop can burn tokens fast — run this periodically to build cost intuition. Pro/Max subscribers use /stats instead.
/cost # API users /stats # Pro/Max subscribers

perf
Visualize daily usage, session history, and usage streaks for Pro/Max subscribers. Shows session count, total time, and usage relative to your plan limits.
/stats
/context
perf
Visualize context window usage as a colored grid, broken down by category: files, conversation history, system prompt, tool results. Shows warning if skills exceed the character budget. Run before deciding whether to /compact.
/context
/usage
perf
Show plan usage limits and rate limit status. Essential for knowing how close you are to hitting your monthly message cap or rate limit before it causes interruption.
/usage
/status
Show Claude Code version, current model, account email, connectivity status, and active MCP server connections. The diagnostic overview for your session environment.
/status
/doctor
Run diagnostics on your Claude Code installation. Checks Node.js version, API key, authentication, network connectivity, and MCP server health. Run this first when something isn't working.
/doctor
/diff
essential
Open interactive diff viewer showing all file changes in the current session. Toggle between full git diff and per-turn diffs. Navigate between files with arrow keys. Run this habitually before every commit.
/diff
/export
optional: [filename]
Export the current conversation as plain text to a file or clipboard. Useful for documentation, post-mortems, sharing session output with teammates, or creating audit trails.
/export /export session-2026-03-20.txt
/copy
Copy the last assistant response to clipboard. If the response contains multiple code blocks, shows a picker to select which block to copy. More precise than manually selecting text.
/copy
/release-notes
View the full Claude Code changelog. Shows what changed in the current version and recent releases. Run after updating to learn about new commands and features.
/release-notes
/insights
advanced
Generate a detailed HTML report analyzing your Claude Code sessions from the past month. Covers usage patterns, costs, tool usage frequency, session history, and workflow efficiency. Run monthly.

MODEL & MODE CONTROL

/model
essentialperf
Switch model mid-session without losing conversation. Use left/right arrows to also adjust effort level. Sonnet for 80% of tasks, Opus for hard problems, Haiku for quick lookups. Switching reduces cost significantly.
/model # interactive picker /model sonnet # balanced, default /model opus # most capable, expensive /model haiku # fastest, cheapest
/fast
perf
args: [on | off]
Toggle fast mode — runs the same model (Opus 4.5) with speed-optimized API settings. Ideal for interactive rapid iteration and live debugging. Turn off when cost efficiency matters more than speed. Note: enabling mid-session re-bills prior context at fast mode rates.
/fast on /fast off
/plan
advanced
Enter plan mode — Claude reads files and analyzes your codebase but cannot make changes. All proposed modifications are presented as plans requiring explicit approval. Toggled via Shift+Tab. Use before production-critical changes.
/plan # or Shift+Tab to cycle modes
/output-style
args: [style]
Switch how Claude formats its responses. Options: Default (balanced), Explanatory (verbose with reasoning shown), Learning (teaching-focused with explanations). Persists for the session.
/output-style # picker /output-style concise /output-style detailed /output-style explanatory
/vim
advanced
Toggle vim keybindings for the prompt input. Full modal editing: Normal/Insert modes, hjkl navigation, word motions, text objects, d/c/y/p operators. Toggle off with /vim again. See the Vim Mode tab for all keybindings.
/vim # toggle on / off
/theme
Change the color theme: light, dark, high-contrast, and colorblind-accessible options. Affects syntax highlighting in code blocks and the overall terminal color scheme.
/theme # opens theme picker

SESSION CONTROL

--continue / -c
essential
Resume the most recent conversation. Equivalent to /resume inside a session. Never lose context when switching tasks.
claude -c
--resume / -r
Resume a session by name or ID. Pass the session name you set with /rename.
claude -r "auth-refactor" claude --resume abc123-session-id
--session-id
Resume or reference a session by its full UUID. Get UUIDs from /insights or session history.
claude --session-id "550e8400-e29b-41d4-..."
--from-pr
Resume sessions linked to a specific GitHub Pull Request number. Useful for continuing work on a PR across sessions.
claude --from-pr 123
--fork-session
Fork into a new session when resuming, rather than continuing in-place. Preserves the original session for later.
claude -r abc123 --fork-session
--worktree / -w
advanced
Create an isolated git worktree and start a session inside it. Changes are isolated from your main branch. Worktrees auto-clean if no changes are made. /batch uses this internally for parallel file operations.
claude -w claude --worktree feature-branch
--add-dir
Add extra working directories to the session. Equivalent to /add-dir inside a session but set at launch. Skills in added directories are auto-loaded.
claude --add-dir ../lib ../apps/web

PRINT MODE (NON-INTERACTIVE / CI/CD)

--print / -p
essential
Send a query and exit — no interactive session. The gateway to scripting Claude Code. Combine with --output-format json for machine-readable pipelines. Supports stdin piping.
claude -p "explain this codebase" > docs.md cat error.log | claude -p "what caused this crash?" claude -p "review code" --output-format json | jq '.result'
--output-format
advanced
values: text · json · stream-json
Control output format in print mode. text (default) for human-readable. json for machine-readable pipelines. stream-json for streaming JSON (Server-Sent Events style), compatible with jq and other tools.
claude -p "analyze" --output-format json claude -p "review" --output-format stream-json claude -p "explain" --output-format text
--input-format
values: text · stream-json
Set the input format for stdin piping. stream-json allows piping JSON-structured input for complex multi-turn scripts.
claude -p --input-format stream-json
--json-schema
Validate the output against a JSON Schema. The response is guaranteed to match the schema or an error is returned. For reliable structured output in pipelines.
claude -p --json-schema '{"type":"object","properties":{...}}' "query"
--max-turns
advancedperf
Cap the number of agentic turns before stopping. Circuit breaker for automated pipelines where runaway execution would be costly. Essential in CI/CD.
claude -p "fix all lint errors" --max-turns 10
--max-budget-usd
perf
Cap spending for this single run. Claude stops when the USD budget is reached, even if the task is unfinished. Best safety net for expensive automated operations.
claude -p "refactor auth module" --max-budget-usd 5.00 claude -p "review PR" --max-budget-usd 1.00 --max-turns 5
--fallback-model
Specify a fallback model to use if the primary model is overloaded or unavailable. Only effective in print mode. Prevents failed CI runs due to temporary API overload.
claude -p "query" --fallback-model sonnet

MODEL & BEHAVIOR

--model
Set the model at launch. Accepts shorthand (sonnet, opus, haiku) or full model string (claude-sonnet-4-20250514, etc.).
claude --model opus claude --model claude-haiku-4-5-20251001
--permission-mode
values: interactive · plan · auto-accept
Start in a specific permission mode instead of the default interactive mode. plan = read-only; auto-accept = approve all tool use automatically.
claude --permission-mode plan claude --permission-mode auto-accept
--agents
advanced
Define custom subagents inline at launch via JSON. Useful for CI/CD pipelines where you want agent behavior defined in the pipeline config rather than committed files.
claude --agents '{"reviewer":{"model":"sonnet","persona":"..."}}'
--agent
Specify a custom agent profile for the session by name. Agent must be defined in .claude/agents/ or ~/.claude/agents/.
claude --agent my-custom-reviewer

PERMISSIONS & TOOLS

--allowedTools
advanced
Tools that run without permission prompts. Space-separated list. Safer alternative to --dangerously-skip-permissions — you get automation without giving Claude carte blanche.
claude -p --allowedTools "Bash(git log:*)" "Read" "Grep" "Glob" \ "Find issues in this codebase"
--disallowedTools
Tools that are completely blocked for this session. Takes precedence over --allowedTools. Use to prevent destructive operations in automated contexts.
claude --disallowedTools "Bash(rm *)" "Edit"
--tools
Restrict which built-in tools are available (whitelist). Claude can only use tools in this list. More restrictive than --allowedTools which controls prompting, not availability.
claude --tools "Bash,Read,Grep"
--dangerously-skip-permissions
advanced
Skip ALL permission prompts — fully autonomous operation. Use ONLY in trusted isolated containers/sandboxes. Never on production systems. Also known as "YOLO mode" (v2.0.68+).
claude --dangerously-skip-permissions \ "refactor all component files" # containers only!

SYSTEM PROMPT & CUSTOMIZATION

--system-prompt
Replace the entire system prompt. Removes all default Claude Code instructions. Use only when you need full control. Mutually exclusive with --system-prompt-file.
claude --system-prompt "You are a security-focused code reviewer only."
--system-prompt-file
Replace the system prompt with contents of a file. Enables version-controlled system prompts shared across the team.
claude --system-prompt-file ./prompts/reviewer.txt
--append-system-prompt
essential
Add custom instructions to the default system prompt without replacing it. The safe choice for most use cases — keeps Claude Code's built-in capabilities while adding your custom rules.
claude --append-system-prompt "Always use TypeScript. Never use any." claude --append-system-prompt-file ./team-rules.txt
--settings
Load additional settings from a JSON file. Merges with existing settings. Useful for environment-specific configurations in CI/CD.
claude --settings ./ci-settings.json

Quick Input Prefixes
Three prefix characters that unlock different input modes instantly
/ — Slash Commands
Type / to see all available commands and skills. The list filters as you type. This is your entry point to /compact, /diff, /model, /plan, and every custom skill in your project.
/ # show all commands /comp # filter — shows /compact /my-skill # invoke a custom skill
! — Bash Mode
perf
Run a shell command DIRECTLY. Bypasses Claude's conversational mode entirely — the command runs immediately in your shell and the raw output is added to conversation context. Tab autocomplete works. Fewer tokens, faster results than asking Claude to run it.
!npm test # run tests, output in context !git log --oneline -5 !cat src/config.ts # read file directly !ls src/ | grep Test # pipeline works
@ — File Reference
perf
Explicitly reference a file or directory in your prompt with tab autocomplete. Tells Claude exactly what to look at, eliminating the search phase. Eliminates 2-3 tool-use turns compared to describing the file location.
@src/auth/LoginForm.tsx fix the validation bug @./tests/auth.test.ts explain the failing test @package.json what version is React?

CLI Flags — Complete Reference
Flags for launching and scripting Copilot CLI. Run copilot help for full reference.
--prompt / -p
essential
Send a single prompt and exit (programmatic mode). Gateway to scripting. Combine with tool flags to allow file modifications in automation.
copilot -p "summarize changes in this PR" copilot -p "fix lint errors" --allow-all-tools

Resume a previous interactive session by name or ID. Can also select from a list.

advanced
Allow all tools without manual approval in programmatic mode. Grants the same file and shell access as the user. Use with caution; only in trusted isolated environments.
copilot -p "update deps" --allow-all-tools
--allow-tool / --deny-tool
advanced
Fine-grained tool permission control. Format: Kind(argument). Deny rules ALWAYS take precedence over allow rules, even when --allow-all is set. :* suffix matches command stem + space (prevents partial matches).
copilot --allow-tool='shell(git:*)' \ --deny-tool='shell(git push)' # Allow all git except destructive push copilot --allow-tool='MyMCP(create_issue)'

new
Launch with experimental features enabled. Once you use /experimental on inside a session, this flag is no longer needed on subsequent launches.
copilot --experimental

advanced

copilot --acp copilot --acp --port 3000

Add MCP servers for a single session without modifying persistent config. Testing new MCP servers or session-specific integrations.
copilot --additional-mcp-config ./mcp-test.json
--disable-builtin-mcps
Disable all built-in MCP servers for this session. Use --disable-mcp-server for a specific server. For controlled/sandboxed environments.
copilot --disable-builtin-mcps copilot --disable-mcp-server github

Token Optimization & Cost Reduction
12 concrete strategies to maximize speed, minimize cost, and get better results from both tools
📊
Cost Reality Check
A typical coding session with Claude Code Opus burns 50K–200K input + 10K–50K output tokens ($0.50–$2.25/session). With Sonnet it's ~5× cheaper ($0.10–$0.45). With Copilot, each interaction uses 1 premium request from your monthly allowance; GPT-4.1 and GPT-5 mini are FREE (0 premium cost). Smart model selection has a direct, measurable impact on your bill.
1
Right Model for Each Task
Don't default everything to the most powerful model. Use Haiku/Sonnet for exploration, reading, and routine edits. Reserve Opus for genuinely hard problems: complex architecture, subtle bugs, multi-step reasoning. Switch back when the hard part is done.
# Claude Code /model haiku # file searches, quick lookups /model sonnet # coding tasks (best value) /model opus # hard problems only # Copilot: free models = 0 premium cost /model gpt-4.1 # free, solid /model gpt-5-mini # free, fastest /model claude-sonnet # balanced, best value
2
Compact Before Overflow
Don't wait for the "context is getting large" warning. Run /compact proactively at 80% with focus instructions. Starting fresh after overflow wastes all accumulated context and forces Claude to re-read everything.
# Check context level first /context # Compact with focus instructions /compact retain the auth flow and DB schema # Copilot auto-compacts at 95% # Manual when you want to compact earlier: /compact
3
Use ! for Direct Shell Commands
Prefixing with ! executes the command directly, bypassing Claude's conversational narration entirely. Claude sees only the raw output — not a description of what it's about to do. Saves 50–200 tokens per command.
# SLOW: Claude narrates and reasons run npm test to check current state # FAST: Direct, just the result in context !npm test !git status !ls src/components/ | grep Button
4
Reference Files Directly with @
Using @filename puts exact file contents in context without Claude spending 2-4 tool-use turns searching for it. In Copilot, also use #issue-number to reference GitHub issues directly.
# Claude Code @src/auth/LoginForm.tsx fix the validation bug @./tests/auth.test.ts explain the failing test # Copilot: also reference GitHub issues #123 implement the feature described here #456 close this bug by fixing the edge case
5
Invest in Project Memory
Every prompt you repeat (your stack, conventions, test framework) is wasted tokens every single session. Project memory files let you say it once and never repeat it. A 30-minute investment pays back in every session forever.
# Claude Code: generate + refine /init # auto-generate CLAUDE.md /memory # edit and add detail # Copilot: manual setup # .github/copilot-instructions.md # Include in both: # - Tech stack and versions # - Coding conventions # - Test framework + how to run # - Build commands # - File naming conventions
6
Plan Before Implementing
Plan mode is read-only — no file writes, no token-burning failed attempts. One good plan with your feedback is worth 3 wrong implementations that each require rollback and retry. Always plan for anything touching production code.
# Claude Code: enter plan mode /plan # or Shift+Tab # Copilot: explicit plan request /plan implement OAuth2 with PKCE for mobile # Review the plan, give feedback # Only then exit plan mode: # Shift+Tab → back to normal "Looks good, proceed with step 1"
7
Restrict Tools in CI/CD
In automated pipelines don't give read-write-execute when you only need analysis. Read-only restrictions prevent accidental state changes and make token usage predictable. Always set --max-turns as a circuit breaker.
# Claude Code: read-only CI analysis claude -p "find security issues" \ --allowedTools "Read" "Grep" "Glob" \ --max-turns 10 \ --max-budget-usd 2.00 # Copilot: fine-grained shell rules copilot --allow-tool='shell(git:*)' \ --deny-tool='shell(git push)' \ --deny-tool='shell(rm:*)'
8
Use Subagents for Focus
One generalist context handling everything produces worse results and more token waste than specialized subagents with focused contexts. A "reviewer" agent that only checks security runs in a smaller, more targeted context window.
# Claude Code: specialized agents # .claude/agents/security.md --- name: security-reviewer model: sonnet --- Focus ONLY on security vulnerabilities. # Copilot: parallel fleet execution /fleet check these 5 components simultaneously # Delegate long tasks to cloud agent & implement the full settings page
9
Monitor Costs in Real Time
A single agentic loop can unexpectedly burn tokens. Build a habit of checking usage regularly. Know when a task is becoming expensive before it's too late to switch strategy.
# Claude Code: health check workflow /cost # how much spent? (API) /stats # usage summary (Pro/Max) /context # context window fullness /usage # near rate limit? # If costs climbing: /compact # compress context /model sonnet # switch cheaper model # Monthly review /insights # 30-day HTML usage report
10
Build Skills for Repeating Tasks
Repeating a 5-sentence complex prompt every day for the same task is wasteful. Package common workflows as skills or custom commands. One /deploy invocation vs. writing deployment steps from scratch every time.
# Claude Code: create a skill # .claude/skills/deploy/SKILL.md --- name: deploy description: Deploy to staging or production --- 1. Run full test suite: npm test 2. Check for uncommitted changes 3. Build: npm run build 4. Deploy: npm run deploy:staging # Invoke: /deploy # Copilot: .github/skills/deploy/SKILL.md # Same structure, invoke: /deploy
11
Prompt Caching (API Users)
For teams using the Claude API directly: prompt caching gives 90% savings on cache reads (0.1× base cost). Cache writes cost 1.25× (5-min) or 2× (1-hour), but each cache hit pays back immediately. Structure prompts to maximize cache hits — static content first, dynamic input last.
# Structure: STATIC front, DYNAMIC end # Static (cached): system prompt, CLAUDE.md, large files # Dynamic (not cached): current task, user question # Minimum cacheable block: 1024 tokens # Copilot equivalent: Copilot Memory # Automatically learns patterns and # reduces repetitive context over time
12
Extended Thinking — Use Wisely
Extended thinking costs more tokens (deeper reasoning = more output). Don't use for routine tasks. Reserve it for genuinely hard problems: architectural decisions, debugging race conditions, complex algorithm design.
# Claude Code: toggle for next turn Alt+T # (run /terminal-setup first) # Good uses for extended thinking: # - System architecture decisions # - Debugging race conditions # - Complex algorithm design # - Understanding confusing legacy code # Don't use for: # - Simple file edits # - Boilerplate generation # - Quick Q&A questions
